# 中文文本对话评测项目

这是一个面向中文大模型产品、训练、评测与数据策略岗位的公开展示项目。

**[在线展示](https://agaresd47.github.io/text-dialogue-benchmark-showcase/)**

## 项目概述

本项目对外呈现为一个中文对话 benchmark 原型。它的核心目标并不是展示零散案例，或给出脱离语境的单一模型排名，而是说明如何从长期积累的真实长对话中识别高价值窗口，并将其蒸馏为可重复测试、可跨模型比较、可继续扩展的评测资产。

当前公开成果包括 `core 8` 展示子集、`full 16` 正式 benchmark、`48` 条基线结果，以及一个包含 `4` 条代表性案例的多评审敏感性检验。项目重点展示的不是样本数量本身，而是从真实长对话到样本构造、评测设计、结果分析与结论收束的完整工作流程。

## 项目定位

本项目重点对应以下能力方向：

- 从真实长对话中识别值得持续研究的高价值片段
- 将长语境交互问题压缩为可公开展示的 benchmark 案例
- 以统一 rubric 与错误分类框架比较不同模型在对话策略层面的稳定差异
- 将评测结果进一步整理为适合产品分析、训练讨论与岗位展示的项目资产

## 核心贡献

我在该项目中主要承担了以下工作：

- 样本蒸馏：从长期积累的真实长对话中筛选可代表隐性需求模式的高信号片段
- 分组设计：将案例按用户状态与交互需求归纳为四类典型用户状态
- 评测设计：为 benchmark 配套统一 rubric 与错误分类框架
- 基线组织：完成 `16` 条案例、`3` 个模型、`48` 条结果的运行、整理与分析
- 补充检验：引入 Gemini 与 Claude 评审，对代表性案例进行多评审敏感性检验
- 展示制作：将 `core 8` 组织为可直接托管的静态展示页，用于公开展示与面试讲述

## 四类典型用户状态

- `B1`：表面求建议，实际求确认
- `B2`：表面做分析，实际在确认是否被理解
- `B3`：表达不显性，需要主动识别信号
- `B4`：已有主见，更需要校准与承接

## 样本筛选原则

本项目未将扩大样本量作为首要目标，而是采用质量优先的筛选策略。当前正式样本主要依据以下标准入选：

- 表达方式具有生活化特征
- 隐性需求具备可识别性
- 好回复与差回复之间存在稳定差异
- 能够较清晰地代表某一类交互问题
- 不依赖大量背景信息即可理解

基于这一原则，当前公开评测集保留 `16` 条正式案例，并从中进一步选出 `core 8` 作为展示子集。该子集更适合用于问题暴露、模型比较与项目讲述。

## 评测口径说明

当前 `full 16` 的公开主结果采用 `ChatGPT` 评审口径。这并不意味着对话任务存在单一真值；不同用户、不同产品团队对于理想回复风格的判断本来就可能不同。

为避免将当前排序误读为唯一正确答案，仓库额外补充了一个包含 `4` 条代表性案例的多评审敏感性检验，并引入以下评审口径：

- `ChatGPT`
- `Gemini`
- `Claude`

这一补充检验表明，模型之间的排序不仅取决于 responder 本身，也会受到评审对理想回复风格偏好的影响。在当前检验中，`B1`、`B2`、`B4` 更容易被判给 `Claude`，`B3` 更容易被判给 `Gemini`。

因此，当前仓库中的排序更适合被理解为特定评审口径下的相对结果，而不是单一标准下的固定结论。

## 当前阶段的主要发现

当前结果显示，三家模型的差异主要不在表层措辞风格，而在于是否能够准确判断用户当下需要怎样的回应方式，包括承接姿态、推进与收束时机、边界感，以及关系连续性的处理。

相较于“谁的总分更高”，本项目更重要的发现是：主流模型普遍已经形成较稳定的默认风格，但在不同用户状态与不同交互情境下，仍然缺乏及时切换回应方式的能力。这一问题比单次排序更接近中文对话产品与行为评测中的真实挑战。

## 岗位相关性

该项目与以下岗位方向具有较强关联：

- **产品方向**：说明默认回复策略不应只有单一风格，而应根据轮次与用户状态动态调整
- **评测方向**：说明 conversation eval 不应默认单一评审就代表唯一正确答案
- **训练方向**：说明高价值数据构造应围绕特定类别持续补强，而不是简单扩大样本量
- **数据策略方向**：说明单轮 prompt 难以暴露连续交互问题，后续轮次往往才是差异真正拉开的阶段

## 后续扩展方向

当前样本主要集中在低到中等情感烈度场景，更多覆盖轻悬空、轻确认、轻校准与轻陪伴等状态。若继续扩展，本项目更优先的方向并不是继续堆叠样本数量，而是沿情感烈度做梯度扩展，以观察同一类隐性需求在更高烈度情境下，模型是否能够切换到不同的承接方式。

更长远的扩展方向，是将上游长对话处理、中游样本构造与下游错误分析逐步系统化，进一步形成可持续扩展的中文对话评测流程。

## 项目要回答的问题

当团队已经拥有大量长周期真实对话材料时，更关键的问题并不是继续积累更多零散观察，而是：

- 哪些长对话片段真正值得研究
- 哪些片段可以被整理为高信号案例
- 同一组案例在不同模型上会稳定暴露什么问题
- 这些问题能否被压缩为一套可复用的错误分类框架，而不是停留在主观印象层面

这个仓库给出的，是这条路线的一个最小可执行版本。
