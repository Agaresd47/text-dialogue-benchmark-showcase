# Results Analysis

## 这轮结果到底说明了什么

最终跑出来的是：

- `16` 条 case
- `4` 个 bucket
- `3` 家模型
- `48` 条 baseline 结果

这套结果已经够回答三个关键问题：

1. 哪些 case 真有区分度
2. 哪些模型更像人话
3. 哪些错误是稳定的 taxonomy 问题，而不是偶发发挥失常

## Core 8 和 Full 16 的关系

`core 8` 不是小号 benchmark，而是最值得讲、最能暴露差异的展示集。

它的作用是：

- 面试里重点讲
- 快速展示 bucket 覆盖
- 让面试官不用先看 16 条全部细节

`full 16` 的作用是：

- 保证 benchmark 不是只挑最漂亮的几条
- 保证每个 bucket 都有 `4` 条
- 给后续扩展和替换留空间

## 哪些 case 最能打

当前最值得拿出来讲的，是：

- `B2-PRODUCT-01`
- `B4-CALIBRATION-01`
- `B3-LIGHT-01`
- `B3-SUSPEND-01`
- `B1-CONFIRM-01`
- `B1-CONFIRM-03`
- `B4-RELATION-01`

这里面最关键的新补位是 `B3-SUSPEND-01`。它把原来 `Bucket 3` 里比较弱的第二个 core 位补强了，让 `Bucket 3` 不再只剩“低能量陪伴”一种 texture。

## 三家模型的结果怎么讲

### gpt-5.4-mini

可以讲成：

- 最像人话
- 最会先接住，再判断，再轻推一步
- 在 `Bucket 2` 和 `Bucket 4` 尤其稳

它的问题不是没理解，而是有时太快进入“帮你处理”，所以会出现轻度：

- 过早建议
- 过于工整
- 微型 action plan 感

### claude-haiku-4-5-bedrock

可以讲成：

- 克制
- 边界感强
- 不太会乱飞

它的典型问题是：

- 太收
- 太保守
- 到 `q15` 容易撤答

也就是说，它更像一个不愿犯错的系统，而不是一个会把对话往前轻轻带一步的系统。

### gemini-3-flash-preview

可以讲成：

- 不是完全看不懂
- 甚至经常概念抓得很快

但它最容易掉进：

- 高位解释
- 说教化
- 过度脑补
- 过说明感

所以它更适合做反例。它最能说明一件事：模型“理解很多”，不等于“适合做对话产品”。

## taxonomy 上最稳定暴露的问题

当前最常见的错误，不是知识错，而是互动策略错：

- `E2.4` 说教化输出
- `E5.2` 下一步过空或过虚
- `E4.1` 过度脑补
- `E3.3` 高位俯视
- `E2.1` 过早建议

这很重要，因为它说明训练端 / 评测端真正需要盯的不是“答没答到”，而是：

- 有没有误判用户要什么
- 有没有选错回应策略
- 有没有把对话产品做成讲课产品

## 为什么这个结果对训练端有价值

因为这套 benchmark 已经在做一件训练端会关心的事：

- 不是评价模型会不会说中文
- 而是评价模型会不会在连续对话里稳定犯某类错误

如果把这套方法扩展到更大的数据库，就可以做：

- 长对话抽取
- 高价值 case 自动发现
- 新旧模型对照
- 短测 / 长测拆分
- taxonomy 驱动的错误复盘

这就是这个项目最值得讲的地方：它不只是一个 demo，而是一个可以继续长成自动化评测手段的雏形。
